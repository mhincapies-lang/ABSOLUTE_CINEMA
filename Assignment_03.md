Compromisos de Trabajo del Grupo

Para el desarrollo del proyecto durante este semestre, el equipo acuerda trabajar bajo un enfoque de trabajo colaborativo utilizando la metodología ágil Kanban, apoyándonos en Jira como herramienta principal para la organización y seguimiento de tareas. Los siguientes compromisos guiarán nuestro trabajo:

1. Uso del tablero Kanban (Jira)

- Todas las tareas del proyecto deberán registrarse en Jira antes de ser realizadas.
- El tablero Kanban se organizará al menos con las columnas: Pendiente (To Do), En Progreso (In Progress), En Revisión (Review) y Terminado (Done).
- Cada integrante será responsable de actualizar el estado de sus tareas en el tablero.
- Ninguna tarea se considerará terminada si no está marcada como Done en Jira.
- Las tareas deben ser lo suficientemente pequeñas para poder completarse en pocos días.

2. Reuniones y sincronización

- El equipo realizará al menos una reunión semanal obligatoria para revisar avances, dificultades y planear el trabajo siguiente.
- Adicionalmente, se podrán realizar reuniones cortas cuando sea necesario para resolver dudas técnicas o de organización.
- Si un integrante no puede asistir a una reunión, deberá avisar con anticipación al grupo.

3. Uso de Git y GitHub

- Cada nueva funcionalidad o cambio importante se realizará en una rama independiente (branch).
- No se realizarán cambios directamente en la rama principal (main).
- Todo cambio deberá pasar por revisión de al menos un integrante del equipo antes de hacer merge.
- Los commits deben tener mensajes claros y descriptivos.
- Antes de subir cambios se debe actualizar el repositorio local para evitar conflictos.
- En caso de conflictos de merge, el responsable de la rama deberá resolverlos.

4. Responsabilidad y cumplimiento de tareas

- Cada integrante asumirá responsabilidad sobre las tareas que seleccione en Jira.
- Si un integrante no puede cumplir una tarea a tiempo, deberá avisar lo antes posible para reorganizar el trabajo.
- Ninguna tarea debe quedar abandonada sin notificación al grupo.

5. Toma de decisiones

- Las decisiones técnicas y organizativas se discutirán en grupo.
- Se buscará llegar a un consenso y, si no es posible, se tomará la decisión por mayoría simple.
- Las decisiones importantes quedarán registradas en el repositorio o en Jira para referencia futura.

Estos compromisos buscan asegurar una organización clara del trabajo, una comunicación efectiva entre los integrantes y un desarrollo ordenado del proyecto durante el semestre.
